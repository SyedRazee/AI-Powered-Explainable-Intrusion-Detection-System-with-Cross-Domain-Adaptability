About Our Project

AI-Powered Network Intrusion Detection System (NIDS)

Project Overview

This is an AI-Powered Network Intrusion Detection System (NIDS) that uses Machine Learning and Deep Learning techniques to detect suspicious activities and cyber attacks in network traffic.

Our system analyzes network traffic in real-time and identifies multiple attack types, such as DDoS attacks, port scanning, malware infections, and other security threats. The system uses advanced ML/DL models that detect threats with high accuracy.

One of the most important features of this system is the use of Explainable AI (XAI) with SHAP (SHapley Additive exPlanations), which helps users understand why and how the model detected a specific attack. This transparency is crucial for security analysts to make informed decisions.

Additionally, our system incorporates Adversarial Defense mechanisms to protect against adversarial attacks that attempt to fool machine learning models by introducing carefully crafted perturbations to network traffic features.

One of the key strengths of our system is Cross-Domain Adaptability, which enables the system to work effectively across different network environments and domains. By providing specialized models for enterprise networks, IoT environments, and cloud infrastructures, the system ensures optimal performance regardless of the network domain.

Key Features

Multiple AI Models

Supports 4 different ML/DL models - CICIDS, IoT-23, TONIOT, and UNSW-NB15. Each model specializes in detecting different types of attacks with high precision and accuracy.

Real-Time Detection

Analyzes network traffic in real-time and instantly detects threats, enabling immediate action to be taken to prevent security breaches.

Explainable AI (SHAP)

Uses SHAP (SHapley Additive exPlanations) to identify which features were most important in attack detection, providing visual feature importance plots for transparency.

Adversarial Defense

Implements surrogate adversarial models to detect and defend against adversarial attacks, including FGSM (Fast Gradient Sign Method) based attacks that attempt to fool ML models.

Batch Processing

Upload CSV files to process multiple records simultaneously, perfect for bulk analysis and large-scale network traffic monitoring.

Comprehensive Dashboard

View system statistics, attack distribution, and detection history through an intuitive visual dashboard with charts and analytics.

Mitigation Suggestions

Provides specific mitigation strategies and security recommendations for each detected attack type, helping administrators take appropriate action.

One-Click Mitigation

Apply mitigation actions instantly with individual "Mitigate" buttons for each detection or use "Mitigate All" to process all detections at once with confirmation dialogs and visual feedback.

Prediction History

Maintains a complete history of all predictions with timestamps, model information, and adversarial attack detection for audit and analysis purposes.

Cross-Domain Adaptability

Supports multiple network domains (Enterprise, IoT, Cloud) through specialized models, allowing the system to adapt to different network environments and maintain high accuracy across domains.

Advanced Filtering System

Comprehensive filtering capabilities in the History section allow users to filter predictions by time range (1 hour, 6 hours, 24 hours, 7 days), model type, and attack type. This enables quick analysis and identification of specific patterns or threats in the detection history.

Supported Models

CICIDS Dataset

Type: Machine Learning Pipeline
Domain: Enterprise Networks
Purpose: General network intrusion detection
Features: Multiple attack categories including DDoS, Port Scan, Brute Force, SQL Injection
Model: Scikit-learn Pipeline (Joblib)
Adversarial Defense: Surrogate adversarial model available
Cross-Domain: Optimized for traditional corporate network environments

IoT-23 Dataset

Type: Gradient Boosting (XGBoost)
Domain: IoT Environments
Purpose: IoT device traffic analysis
Features: Specialized for IoT network security threats
Model: XGBoost Classifier
Adversarial Defense: Surrogate adversarial model available
Cross-Domain: Specialized for smart devices and IoT sensor networks

TONIOT Dataset

Type: Machine Learning Pipeline
Domain: IoT & Cloud Infrastructures
Purpose: IoT and cloud environment security
Features: Comprehensive IoT attack detection
Model: Scikit-learn Pipeline (Joblib)
Adversarial Defense: Surrogate adversarial model available
Cross-Domain: Designed for IoT devices and cloud-based services

UNSW-NB15 Dataset

Type: Deep Learning (Neural Network)
Domain: General Networks
Purpose: Advanced threat detection
Features: 10 attack categories including Analysis, Backdoor, DoS, Exploits, Fuzzers, Generic, Normal, Reconnaissance, Shellcode, Worms
Model: Keras/TensorFlow Neural Network
Adversarial Defense: Integrated adversarial tracking
Cross-Domain: Versatile model for various network environments

NSL-KDD Dataset

Type: Machine Learning (XGBoost)
Domain: General Network Intrusion Detection
Purpose: Standard benchmark for intrusion detection systems
Features: 4 key features including service, flag, src_bytes, and diff_srv_rate for efficient attack classification
Model: XGBoost Classifier
Attack Types: Dos, Probe, R2L (Remote to Local), U2R (User to Root), Normal
Cross-Domain: Widely used benchmark dataset for evaluating IDS performance across different network environments
